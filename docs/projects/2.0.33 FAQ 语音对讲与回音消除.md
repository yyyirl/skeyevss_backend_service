## 1. 对讲与单向直播的区别

对讲是 **双向、低延迟、需回声消除（AEC）** 的会话；仅把麦克风流推到设备 **不等于** 完整对讲能力。架构上可能涉及 **WebRTC、SIP、RTP 双向、或厂商私有协议**。

---

## 2. 典型问题

- **单方无声**：编解码不一致（如平台 Opus、设备仅 G.711）、RTP 单通、防火墙阻断 **反向 RTP**；
- **延迟大**：经过多层转码或缓冲过大；
- **啸叫/回音**：播放端扬声器与麦克风的 **声学回路** 未做 AEC；浏览器端 **getUserMedia** 约束与 AEC 开关需检查；
- **断续**：抖动缓冲过小或 **丢包** 严重。

---

## 3. NAT 与端口

对讲往往动态协商 **第二个 RTP 会话**；与点播类似，需保证 SDP 地址与端口对双方可达。

---

## 4. 设备侧

- 确认设备 **对讲通道** 已启用、未与 **报警输出** 等资源冲突；
- 部分设备要求 **先建立预览** 再启动对讲；
- 固件 bug 会导致 **长时间对讲后内存泄漏**，需厂商补丁。

---

## 5. 排障步骤

1. **网络层**：双向 UDP 连通；
2. **编码层**：抓 RTP payload type 与采样率；
3. **应用层**：信令是否 **200/ACK** 完整；
4. **客户端**：换耳机排除回音，验证 **是否浏览器限制**。
