# Skeyevss FAQ：灰度发布与流量切换

[试用安装包下载](https://www.openskeye.cn/releases) | [SMS](https://github.com/openskeye/go-vss/releases/tag/V1.0.6) | [在线演示](https://showcase.openskeye.cn/)

**项目地址**：[https://github.com/openskeye/go-vss](https://github.com/openskeye/go-vss)

---

## 1. 目标

新版本 **先小流量验证**，指标正常再全量；出问题 **快速回滚**。对 SIP 类系统，还要考虑 **会话粘性** 与 **注册状态**。

---

## 2. HTTP

- **Ingress/网关** 按 **权重、Header、Cookie** 分流；
- 监控 **错误率、延迟、5xx** 对比基线；
- **特性开关** 可在进程内再细分功能。

---

## 3. SIP/媒体面难点

- UDP **无连接**，LB 哈希需基于 **五元组或 Call-ID**；
- **长时 INVITE** 中途切换版本可能 **状态不一致**；
- 实践中常 **维护窗口 + 排空** 优于 **在线热切 SIP**。

---

## 4. 数据库迁移

**Expand-Contract** 模式：**先兼容读写 → 切流量 → 删旧列**。避免新旧版本 **同时假设不同 schema**。与《版本升级数据迁移与回滚》对照。

---

## 5. 观测

- **业务 KPI**：注册成功率、点播成功率、录像完整率；
- **资源**：CPU、GC、队列深度；
- **对比**：灰度发布 Pod vs 稳定 Pod。

---

## 6. 回滚

**一键回滚** 需 **镜像 tag 固定、配置版本化**；数据库 **不可逆变更** 慎做或 **双写期** 拉长。

---

灰度发布 **不是只给 Web 用的**；视频平台要把 **SIP 状态机** 纳入发布策略，否则「HTTP 没问题但全掉线」。
